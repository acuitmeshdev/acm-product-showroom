# Product Flyer Showroom

Interactive product showroom built with Astro for Acuitmesh solutions.

This project supports **3 brochure modes** per product:

1. **`markdown`** — render a rich product page from frontmatter + markdown body
2. **`html`** — embed a standalone interactive HTML flyer in an iframe
3. **`pdf`** — embed and download a PDF brochure

---

## Tech Stack

- Astro 6
- React (for selected interactive components)
- Tailwind CSS v4
- Astro Content Collections (`src/content/products/*.md`)

## How Routing Works

- Homepage product cards are generated from `src/content/products/*.md`
- Product pages are generated by `src/pages/products/[slug].astro`
- Final route is: `/products/<slug>`

Example:
- `src/content/products/strategic-ai.md` → `/products/strategic-ai`

## Product Schema (Supported Fields)

Defined in `src/content.config.ts`:

- `title: string`
- `tagline: string`
- `description: string`
- `logo: string` (public path, e.g. `/products/my-product/logo.svg`)
- `heroImage?: string`
- `websiteUrl: string` (must be a valid URL)
- `brandColor: string` (hex, default `#6366f1`)
- `brochureType: 'markdown' | 'html' | 'pdf'`
- `brochureFile?: string` (required for `html` / `pdf` mode)
- `features?` (markdown mode)
- `specs?` (markdown mode)
- `pricing?` (markdown mode)
- `gallery?` (markdown mode)
- `videoUrl?` (markdown mode)
- `formspreeId?` (markdown mode)
- `category: string`
- `tags: string[]`
- `publishedAt: date`
- `order: number` (controls card order on homepage)

## Add a New Product

### 1) Create product content file

Create a new markdown file:

`src/content/products/<slug>.md`

Use one of the templates below based on `brochureType`.

### 2) Add static assets

Create folder:

`public/products/<slug>/`

Put logo and brochure files there.

### 3) Run local validation

```bash
npm install
npm run build
```

If build succeeds, product should appear on homepage and have a generated route.

## Templates by Brochure Type

### A) `markdown` product template

```md
---
title: "My Product"
tagline: "Fast secure platform"
description: "A short product description."
logo: "/products/my-product/logo.svg"
heroImage: "/products/my-product/hero.png"
websiteUrl: "https://example.com"
brandColor: "#0ea5e9"
brochureType: "markdown"
category: "Platform"
tags: ["cloud", "security"]
publishedAt: 2026-03-12
order: 3

features:
	- title: "Feature One"
		description: "What it does"
		icon: "sparkles"

specs:
	- label: "Uptime"
		value: "99.9%"

pricing:
	- plan: "Starter"
		price: "$29"
		period: "/month"
		features:
			- "Basic support"
			- "10 users"
		highlighted: false
---

Optional markdown body content here.
```

### B) `html` product template

```md
---
title: "My Interactive Product"
tagline: "Interactive showcase"
description: "Product with standalone HTML flyer."
logo: "/products/my-interactive-product/logo.svg"
websiteUrl: "https://example.com"
brandColor: "#f97316"
brochureType: "html"
brochureFile: "/products/my-interactive-product/flyer.html"
category: "SaaS"
tags: ["interactive", "ai"]
publishedAt: 2026-03-12
order: 4
---
```

Then create:
- `public/products/my-interactive-product/flyer.html`

### C) `pdf` product template

```md
---
title: "My PDF Product"
tagline: "PDF brochure"
description: "Product with PDF brochure."
logo: "/products/my-pdf-product/logo.svg"
websiteUrl: "https://example.com"
brandColor: "#7c3aed"
brochureType: "pdf"
brochureFile: "/products/my-pdf-product/brochure.pdf"
category: "Consulting"
tags: ["pdf", "brochure"]
publishedAt: 2026-03-12
order: 5
---
```

Then create:
- `public/products/my-pdf-product/brochure.pdf`

## Deployment Notes (GitHub Pages)

This project is static (`output: 'static'`) and can be deployed to GitHub Pages.

Before deploying, configure in `astro.config.mjs`:

- `site`: your GitHub Pages domain
- `base`: repo path (e.g. `/product-flyer` for project pages)

Recommended deployment: GitHub Actions builds from source and publishes `dist/`.

## Troubleshooting

- **Product not showing on homepage**
	- Check file is in `src/content/products/`
	- Check valid frontmatter (especially `publishedAt`, `websiteUrl`, `brochureType`)
	- Check `order` value

- **HTML/PDF page shows fallback message**
	- Ensure `brochureFile` is set
	- Ensure file exists in `public/products/<slug>/`

- **Build fails with content schema errors**
	- Compare your frontmatter with schema in `src/content.config.ts`
